IP Addresses: How They Work, The Types, and Their Uses

Introduction

IP addresses, short for Internet Protocol addresses, are unique, numerical identifiers for computers, tablets, phones, servers, and anything connected to Wi-Fi. IP addresses serve two purposes: they identify devices on the network and provide the device’s location.

Structure

There are two types of IP addresses that are used on the internet today: IPv4 and IPv6

  • IPv4:
    • The most widely used format
    • Made up of 4 sets of numbers (0-255) separated by periods
    • Supports 4,294,967,296 unique addresses
  • IPv6:
    • Developed to combat limitations of IPv4
    • Uses a 128-bit format with 8 groups of 4 hexadecimal numbers (0-9, A-F)
    • 340 undecillion unique IP addresses

Functions

  • Location: Every device on a network is assigned an IP address so routers can find the best possible path to or from that device.
  • Identification: An IP address uniquely identifies each device on the internet, enabling communication.

Types

There are two main types of addresses, Private and Public.

  • Public Addresses: Accessible directly from the internet. Used for communication and routing between devices and networks.
  • Private Addresses: Used within homes, schools, and businesses and are not accessible from the internet.

Static vs Dynamic IPs

  • Static IP Addresses:
    • Remain constant and are set manually to a device.
    • Ideal for servers and websites requiring a fixed address.
  • Dynamic IP Addresses:
    • Change every time a device connects to a network.
    • Assigned automatically by DHCP without manual intervention.

Classes of IPv4 Addresses

IPv4 Addresses are grouped into five classes, A through E, based on their size and use case:

  • Class A: Large networks (e.g., Amazon, Google)
  • Class B: Medium-sized networks (e.g., Universities)
  • Class C: Small networks (e.g., Small businesses)
  • Class D: Multicasting (e.g., Streaming services)
  • Class E: Research and experimental purposes

DHCP

DHCP is a protocol that automatically assigns IPs to devices on a network. Each IP is assigned for a limited lease time before being stored or reassigned. This simplifies IP management and prevents conflicts.

Subnetting

Subnetting divides a large network into smaller subnetworks, improving management, performance, and security. A subnet mask determines which part of an IP represents the network and host ID.

NAT

NAT (Network Address Translation) maps multiple private IPs to a single public IP. It conserves IPs, enhances security, and supports private networks.

  • Static NAT: One-to-one mapping of private to public IPs.
  • Dynamic NAT: Chooses available public IPs dynamically.
  • PAT: Maps several private IPs to one public IP using port numbers.

Conclusion

IP addresses are the backbone of the internet. They enable communication and data routing across networks, making them essential for IT and network management.
